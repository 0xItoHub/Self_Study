Javaにおける「ボクシング（Boxing）」と「アンボクシング（Unboxing）」は、プリミティブ型（基本データ型）とラッパークラス（オブジェクト型）の間で自動的に変換が行われる仕組みです。

## ボクシング（Boxing）
プリミティブ型（例えば `int` や `boolean` など）を対応するラッパークラス（`Integer` や `Boolean` など）のオブジェクトに自動的に変換することを指します。これは、Java 5以降に追加された機能で、プリミティブ型をオブジェクト型として扱う場面で便利です。

### 例:
```java
int num = 10;          // プリミティブ型のint
Integer obj = num;     // ボクシング: intからIntegerに自動変換
```
上記の例では、`int` 型の `num` が `Integer` 型の `obj` に自動的に変換されています。これがボクシングです。

## アンボクシング（Unboxing）
ラッパークラスのオブジェクトをプリミティブ型に自動的に変換することを指します。これもJava 5以降に導入され、オブジェクト型からプリミティブ型として値を取り出す際に便利です。

### 例:
```java
Integer obj = 20;     // ボクシング: プリミティブ型の20をIntegerに変換
int num = obj;        // アンボクシング: Integerからintに自動変換
```
この例では、`Integer` 型の `obj` が `int` 型の `num` に自動的に変換されています。これがアンボクシングです。

## 自動ボクシングとアンボクシングの例
```java
List<Integer> list = new ArrayList<>();
list.add(5);  // ここでボクシングが行われる（intからInteger）
int num = list.get(0); // ここでアンボクシングが行われる（Integerからint）
```
`list.add(5)` の部分では、`int` 型の 5 が `Integer` オブジェクトに自動的に変換（ボクシング）され、`list.get(0)` の部分では、`Integer` オブジェクトが `int` に自動的に変換（アンボクシング）されます。

## 注意点
- **NullPointerException:** アンボクシング時にラッパークラスのオブジェクトが `null` の場合、`NullPointerException` が発生します。例えば、次のコードは例外を発生させます:
  ```java
  Integer obj = null;
  int num = obj;  // ここでNullPointerExceptionが発生
  ```

## まとめ
- **ボクシング:** プリミティブ型 → ラッパークラスのオブジェクト型に自動変換。
- **アンボクシング:** ラッパークラスのオブジェクト型 → プリミティブ型に自動変換。

これらの機能は、プリミティブ型とオブジェクト型の間の変換を容易にし、コードの可読性と使いやすさを向上させるために設計されています。
